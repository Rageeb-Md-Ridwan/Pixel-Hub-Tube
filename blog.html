Discuss the scope of var, let, and const

Var:

Function-scoped: Variables declared with var are function-scoped, meaning their scope is limited to the function in which they are declared.
Hoisting: var declarations are hoisted to the top of their function or global context, which means you can access the variable before it's declared, but it will have an initial value of undefined.

Let:

Block-scoped: Variables declared with let are block-scoped, meaning their scope is limited to the block (enclosed by curly braces) in which they are defined.
No hoisting: let declarations are not hoisted to the top of the block; they are only accessible after the point of declaration.

Const:

Block-scoped: Like let, const is block-scoped.
Constant value: Variables declared with const must be assigned a value at the time of declaration and cannot be reassigned. The value remains constant throughout the block scope.

Tell us the use cases of null and undefined

undefined:

Default value: Variables that are declared but not assigned a value are automatically assigned the value undefined.
Function parameters: When a function is called with fewer arguments than declared parameters, the missing parameters are undefined.

null:

Explicit absence of value: It is a value that represents the intentional absence of any object value.
Used for assignments: Developers may explicitly set a variable to null to indicate that it has no meaningful value.
Comparison: null can be used to check if a variable has been intentionally set to represent no value.

What do you mean by REST API?

REST API (Representational State Transfer API) is an architectural style for designing networked applications. It relies on a stateless communication between client and server, where communication is made through standard HTTP methods (GET, POST, PUT, DELETE). RESTful APIs use standardized, resource-based URIs (Uniform Resource Identifiers) to perform operations on resources (data entities), and data is often exchanged in lightweight formats like JSON. REST is known for its simplicity, scalability, and ease of integration, making it a widely adopted choice for building web services.
